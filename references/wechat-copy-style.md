# WeChat Copy Style

Use this reference when generating WeChat Official Account copy for Juhuo AI event promotion.

## Positioning

The WeChat article should feel like a campus AI community invitation, not an internal activity plan. It should be complete enough for public release, warm enough for students to want to come, and structured enough for readers to scan.

## Recommended Structure

Use this structure by default:

```markdown
# Title

Opening visual line or subtitle, optional.

## 导语

Warm invitation. Explain why this event exists and why now.

Highlighted sentence / golden line.

## 01 活动初衷

Translate background and purpose from the plan into student-facing language.

## 02 适合谁来

Use friendly participant cards or short blocks:
- 路演项目组
- 自由组队成员
- 普通观众 / 兴趣小白

## 03 路演 / 报名规则

Use checklist language. Keep it clear, not stern.

## 04 活动流程

Use timeline or concise agenda.

## 时间与地点

List time, location or location uncertainty, registration method.

## 结尾邀请

Close with a concrete scene and CTA.
```

## Voice

Prefer:

- "我们想邀请你来现场聊聊"
- "不是答辩，也不是正式评审"
- "把真实项目拿出来，一起找路径、找队友、找反馈"
- "哪怕你只是好奇，也可以来旁听"

Avoid:

- "为进一步提升活动连续性"
- "形成初步沉淀"
- "赋能生态闭环"
- "核心能力杠杆"
- Excessive "重磅", "爆款", "顶级", "绝佳机会"

## Title Patterns

Good title patterns:

- `邀请函｜炬火 AI 实践月首发：带上你的项目，我们聊聊 AI 怎么落地`
- `这周日，来现场看 AI 如何从工具箱变成真实项目`
- `炬火 AI 实践月启动：寻找正在被 AI 赋能的真实项目`

Keep titles specific. Mention the event identity, audience, or concrete action.

## Golden Lines

Use 1-3 highlighted lines in WeChat copy. They should summarize the shift or invitation.

Examples:

- `这一次，我们想把视线从"AI 工具怎么用"转向"AI 项目怎么做"。`
- `这里没有正式评委，只有真实项目、真实问题和愿意一起支招的人。`
- `带上项目，或者带上好奇心，都可以来。`

Do not create too many golden lines. Too many highlights make the article feel noisy.

## Participant Section

Write participant categories as human invitations, not administrative definitions.

For each category, include:

- Who they are.
- Why they should come.
- What they can do at the event.

Example:

```markdown
**路演项目组**

如果你的项目已经接入 AI，或者正准备把 AI 放进某个真实场景，欢迎带来现场做一次轻量路演。你可以展示进展，也可以直接说出需求：缺技术、缺产品、缺队友，都可以摆到桌面上聊。
```

## Rules Section

Use checklist-like language.

Prefer:

- `AI 相关项目优先`
- `路演名额最多 5 组`
- `如果报名超过名额，将根据项目情况和报名顺序协调顺延`

Avoid sounding punitive. Do not write like a formal review notice.

## Agenda Section

Use the public agenda from the fact table. Exclude staff setup time.

If timing is flexible, add:

```markdown
具体节奏会根据现场情况微调。
```

## Time And Location

If the location is confirmed, write it directly.

If the location is tentative, write:

```markdown
地点：以后续报名群通知为准
```

Do not replace tentative facts with remembered or previous-event locations unless the user explicitly supplies an update.

## WeChat HTML Note

This reference is for copy only. If the user asks for WeChat HTML layout, use static WeChat-compatible HTML with mostly inline styles. Do not rely on animation, JavaScript, external CSS, pseudo-elements, or remote assets.
